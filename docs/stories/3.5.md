# Story 3.5: Validation-speed proof run

**Status**: Draft
**Epic**: 3 — Products & Validation
**Depends on**: 3.1, 3.2, 3.3

## Story

As the site owner, I want to measure how long it actually takes to publish a new
`validating` product, so that AP-12's ½-day-validation-gate premise (the site as its
instrument) is proven, not assumed.

## Acceptance criteria

1. A new `validating` product is published (real product from the backlog, e.g. one
   named in spec Appendix A's future-records list, or a fixture if none is ready) by
   creating exactly one content record file — no code change (AC-2).
2. The elapsed content-authoring time (writing the record, not including any one-time
   setup) is measured and is under 30 minutes (AC-10).
3. The result (pass/fail, actual time) is recorded (e.g. as a note in this story file
   or a short entry in `docs/architecture.md`'s log) so the claim is falsifiable, not
   just asserted.

## Tasks / subtasks

- [ ] Pick the product to validate (owner's call — do not invent product content).
- [ ] Time the authoring of its record file, start to publish-ready.
- [ ] Record the result against AC-10.

## Dev notes

- This story only has a "done" state once a real timing measurement exists — a
  hypothetical estimate does not satisfy AC-10.
