# Story 3.1: Product content type + validation

**Status**: Draft
**Epic**: 3 — Products & Validation
**Depends on**: 1.1 (schema pipeline already defines this schema; this story
authors real records)

## Story

As the site owner, I want a `product` content type covering the full
validating/live/sunset lifecycle, so that each product's landing page can be built
from one record file (AP-12).

## Acceptance criteria

1. `product` records validate against spec §6.2, including the two additive fields
   from `docs/architecture.md` D-10/D-11: `unlisted: boolean` (optional, default
   `false`) and `featured: boolean` (optional, default `false`).
2. `status: sunset` records require `sunset.date` and `sunset.note`
   (`sunset.successor` optional) — build fails if a `sunset` record lacks them.
3. `cta.kind` is restricted to `signup | marketplace | purchase | contact`;
   `platforms` entries restricted to the enum in spec §6.2; `pricing` restricted to
   `free | freemium | paid | tbd`.
4. At least one real or fixture record per lifecycle state (`validating`, `live`,
   `sunset`) exists to feed Stories 3.2–3.4.

## Tasks / subtasks

- [ ] Confirm/extend the `product` schema module from Story 1.1 against real
      content, including the conditional-required `sunset` block.
- [ ] Author fixture/real records for each lifecycle state.

## Dev notes

- See `docs/architecture.md` §9 D-10/D-11 for why `unlisted`/`featured` are present

  field, not a reinterpretation of the schema.
