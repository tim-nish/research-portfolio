# Story 2.2: Writing index page

**Status**: Draft
**Epic**: 2 — Writing System
**Depends on**: 2.1

## Story

As a reader (P1 or P4), I want a unified writing index mixing canonical and external
articles, so that I can find everything the owner has written regardless of where
it's hosted.

## Acceptance criteria

1. `/writing/` renders, per spec §7.3: one-line editorial stance (AP-10-flavored:
   writing grounded in built systems and measured results); unified
   reverse-chronological list mixing canonical + external, each entry showing title,
   date, summary, language badge (EN/JA), platform badge (for external), topic tags;
   external entries link directly out (new tab acceptable), canonical entries link
   on-site.
2. A filter affordance by language (EN/JA) and by canonical/external exists, simple
   and JS-optional (e.g., separate anchor-based views are acceptable per spec).
3. Page readable with JS disabled; filters are progressive enhancement only.
4. SEO metadata present (FR7); this is an index page, no per-entity JSON-LD required
   here (that lives on `/writing/<slug>/`, Story 2.3).

## Tasks / subtasks

- [ ] Build `/writing/` template: unified list, badges, JS-optional filter views.

## Dev notes

- "Simple, JS-optional" filtering is explicitly spec-sanctioned to be as basic as
  separate anchor views (§7.3) — do not over-build this into a JS-required faceted
  search; that would violate NFR1.
