# Story 2.4: Syndication conventions & feed inclusion

**Status**: Draft
**Epic**: 2 — Writing System
**Depends on**: 2.1, 1.9 (feed generator)

## Story

As the site owner, I want syndicated copies to point back here and the feed to
include articles, so that canonicality is never split (AP-6) and followers see all
writing, including JP output.

## Acceptance criteria

1. The editorial checklist in `docs/content-guide.md` documents the `syndication`
   field as the enforcement mechanism: any syndicated copy (e.g. on dev.to) must carry
   `rel=canonical` back to this site — enforced editorially, per spec §8.2 ("enforced
   editorially; the article record's syndication field is the checklist"), not
   mechanically (the syndicated copy lives on a platform this repo doesn't control).
2. `/feed.xml` (Story 1.9's generator, parameterized for exactly this) now includes
   canonical articles as full items; external-mode articles are included as
   link-only items (spec §8.3 recommends this "so followers see JP output too").
3. Feed re-validates after this change.

## Tasks / subtasks

- [ ] Extend the Story 1.9 feed generator to pull from the now-populated `article`
      registry (canonical = full item, external = link-only item).
- [ ] Add the syndication checklist note to `docs/content-guide.md` if not already
      present.

## Dev notes

- No code can force an external platform to add `rel=canonical` — this is
  editorial process, not a build check. Don't build a scraper/verifier for this; it's
  out of scope and not requested by the spec.
