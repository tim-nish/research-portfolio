# Story 4.2: Per-pain / comparison SEO pages (gated)

**Status**: Not started — awaiting go/no-go
**Epic**: 4 — Compounding SEO
**Depends on**: Epic 3 (`product` records and their `pain` field)

## Gate

Do not start without an explicit go decision backed by search data per product/pain
(spec §10's trigger discipline).

## Story

As a searcher with a specific pain not exactly matching one product's primary
phrasing, I want programmatic SEO pages under `/products/…`, so that more of the
long-tail search demand for that pain finds the right product.

## Acceptance criteria (once un-gated)

1. New pages live under the reserved `/compare/…` or per-pain `/products/…` URL space
   (spec §7.1) without colliding with any existing `/products/<slug>/` route.
2. Each page is generated from data already on `product` records (`pain`, `summary`)
   plus new comparison-specific fields — any new field is additive per §6.2's
   allowance, not a repurposing of existing fields.
3. This remains within AP-10 (evidence-gated content) — no generic
   comparison/tutorial content invented to fill the page; content must derive from
   the owner's actual product and real comparison points.

## Tasks / subtasks (once un-gated)

- [ ] Confirm go decision and record the evidence (which product, which pain
      variants, what search data supports it).
- [ ] Design the specific field additions needed (comparison targets, alternate pain
      phrasings) before building — this is a design task, not assumed here.
