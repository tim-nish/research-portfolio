# Story 3.3: Products index page

**Status**: Draft
**Epic**: 3 — Products & Validation
**Depends on**: 3.1

## Story

As a visitor browsing what the owner has built, I want a products index grouped by
lifecycle, so that I can find live tools while retired ones stay visible for
transparency.

## Acceptance criteria

1. `/products/` renders per spec §7.3: one-paragraph framing (small, focused tools,
   built for researchers unless stated); live products as cards (title,
   pain-derived tagline, platform badges, CTA link); validating products listed only
   if `unlisted: false` (Story 3.1's added field — default listed); retired section
   (opt, collapsed) showing sunset products with dates.
2. This page uses the site's global chrome (full nav) — the reduced-chrome exception
   from Story 3.2 applies only to `/products/<slug>/`, not the index (spec §7.2).
3. Retired products remain public and listed (deliberate transparency choice per
   §6.4 — "kept public deliberately") — this story must not add any mechanism to
   fully hide a sunset product, only to collapse its section visually.

## Tasks / subtasks

- [ ] Build `/products/` template with the three grouped sections above.
- [ ] Wire the `unlisted` filter for validating products.

## Dev notes

- Do not confuse `unlisted` (validating-only stealth opt-out) with any notion of
  hiding sunset products — the spec explicitly wants sunset products visible.
