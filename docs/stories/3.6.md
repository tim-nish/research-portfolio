# Story 3.6: Newsletter issue archive (optional)

**Status**: Done
**Epic**: 3 — Products & Validation
**Depends on**: 1.7

## Story

As a newsletter subscriber or prospective subscriber, I want to see past issues, so
that I can gauge value before signing up.

## Acceptance criteria

1. `newsletter-issue` records validate against spec §6.2: `slug`, `number`, `date`,
   `subject`, `summary`, optional archived body or link to ESP-hosted archive.
2. `/newsletter/`'s archive slot (built empty-safe in Story 1.7) now renders a list of
   issues when records exist — no changes to Story 1.7's page structure beyond
   populating what was already an empty-safe block (NFR8).
3. This story is explicitly optional per spec §6.2/§10 — it may be skipped entirely
   for Phase 3 without blocking Epic 3's definition of done.
