# Story 3.2: Product landing page template

**Status**: Draft
**Epic**: 3 — Products & Validation
**Depends on**: 3.1, 1.7 (newsletter capture), 1.8 (SEO/JSON-LD)

## Story

As a prospect arriving from a search for a specific pain (P3, journey J2), I want a
standalone product landing page that converts without needing any other site context,
so that the page itself is the SEO instrument for that pain.

## Acceptance criteria

1. `/products/<slug>/` renders with **reduced chrome** (wordmark-only header, no full
   nav — spec §7.2's deliberate exception; footer retains capture + identity) for
   `validating` and `live` states.
2. Block order for `validating`/`live` (spec §7.3): H1 derived from `pain`, subhead
   from `summary`; primary CTA (from `cta`) above the fold; body (problem → how it
   works → proof → FAQ — template accommodates all four, renders only sections with
   content); platform badges + secondary links; footer capture reworded for context
   ("get notified about what I build next").
3. `sunset` state replaces blocks 2–4 per §6.4: CTA area shows the sunset note, an
   optional successor pointer, and the newsletter capture — exercised via the Story
   3.1 sunset fixture (AC-8).
4. H1/title/meta description are literally driven by `pain` (FR10, spec §8.2 "the
   page is the SEO instrument for that pain") — not a generic product-name headline.
5. `SoftwareApplication` JSON-LD present (spec §8.2).
6. Page stands alone: no assumption that the visitor has seen any other page (AC-9's
   "landing page must stand alone" requirement) — verified by checking the page
   renders full context (what it is, how to get it) without relying on global nav.
