# Story 2.3: Canonical article page template

**Status**: Draft
**Epic**: 2 — Writing System
**Depends on**: 2.1, 1.7 (shared CTA block), 1.8 (shared SEO/JSON-LD component)

## Story

As a reader who followed a link to a canonical article, I want a full article page
with related-entity links and the standard funnel CTA, so that I can go deeper into
the owner's work or subscribe (journey J1/J4).

## Acceptance criteria

1. `/writing/<slug>/` renders (canonical-mode records only — external-mode records do
   not get a detail page, per spec §7.1): title, date (+ `updated` if present), body,
   topic tags, syndication note (opt: "also on dev.to" style, from `syndication`),
   related entities, standard CTA block (reused from Story 1.7).
2. `deprecated` article state renders its §6.4 banner (exercised via fixture if no
   real deprecated article exists yet — AC-8).
3. SEO metadata + `Article` JSON-LD present (FR7, spec §8.2).
4. Page readable with JS disabled.

## Tasks / subtasks

- [ ] Build `/writing/<slug>/` template (canonical mode only; attempting to render an
      external-mode record here should be a build-time impossibility, not a runtime
      guard — routes are generated only for canonical records).
- [ ] Implement the `deprecated` banner.

## Dev notes

- Route generation must only emit `/writing/<slug>/` for `mode: canonical` records —
  this keeps AP-6 ("external entries do NOT get a detail page") true by construction,
  not by a conditional check that could silently pass through.
