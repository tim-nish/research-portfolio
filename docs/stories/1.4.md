# Story 1.4: Benchmarks curated view

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.3

## Story

As a fellow researcher evaluating credibility, I want a dedicated benchmarks view, so
that I can find leaderboards and datasets without wading through all project kinds.

## Acceptance criteria

1. `/benchmarks/` renders a one-paragraph stance on evaluation engineering as a focus
   area (spec §7.3).
2. Below it, `project` records with `kind: benchmark` render as rich cards: title,
   summary, leaderboard link emphasized, dataset links, a citation-availability
   indicator; each card links to `/projects/<slug>/`.
3. This is a filtered view over the same `project` registry used by Story 1.3 — no
   separate content type or duplicate record (spec §6.1 note: "Benchmarks are not a
   separate type").
4. `Dataset`-bearing `CreativeWork` JSON-LD is emitted on the underlying
   `/projects/<slug>/` detail page for benchmark projects with dataset links (spec
   §8.2) — not on `/benchmarks/` itself, which is an index page.

## Tasks / subtasks

- [ ] Build `/benchmarks/` template filtering the project registry by
      `kind === "benchmark"`.
- [ ] Add the dataset-bearing JSON-LD emission to the project-detail template from
      Story 1.3 (shared code path, not duplicated).

## Dev notes

- Exact schema.org shape for "`Dataset` (or `Dataset`-bearing `CreativeWork`)" is left
  to implementer discretion (spec §8.2 phrasing is deliberately loose) — pick one
  consistent shape and apply it uniformly; not tracked as an open question since the
  spec explicitly allows either.
