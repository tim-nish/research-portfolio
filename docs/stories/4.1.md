# Story 4.1: Topic pages from tags (gated)

**Status**: Not started — awaiting go/no-go
**Epic**: 4 — Compounding SEO
**Depends on**: Epic 1 (topics metadata already collected on all types since Phase 1)

## Gate

Do not start without an explicit go decision backed by search/analytics data showing
topic-page demand (spec §10's trigger discipline). This story exists to reserve scope
and design, not to schedule work.

## Story

As a searcher looking for everything the owner has published on a given topic, I want
a `/topics/<tag>/` page, so that I can browse across projects/publications/articles by
subject.

## Acceptance criteria (once un-gated)

1. `/topics/<tag>/` renders all records (any type) carrying that tag in their
   `topics` field, reusing metadata already collected since Phase 1 (spec §6.3: "in
   earlier phases they are metadata only").
2. URL space `/topics/<tag>/` was already reserved and not squatted by any earlier
   phase (spec §7.1) — verify no conflicting route exists before building.
3. Adding this feature requires no changes to Epics 1–3's schemas (AP-9) — only new
   pages/views over existing data.

## Tasks / subtasks (once un-gated)

- [ ] Confirm go decision and record the evidence that triggered it.
- [ ] Build `/topics/<tag>/` as a filtered view (same pattern as Story 1.4's
      benchmarks view).
