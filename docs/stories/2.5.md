# Story 2.5: Backfill external Zenn records + home "recent writing" wiring

**Status**: Draft
**Epic**: 2 — Writing System
**Depends on**: 2.1, 1.6 (home page's recent-writing block, built empty-safe)

## Story

As the site owner, I want existing Zenn output registered as external-mode records and
the home page's recent-writing block wired to real data, so that the hub actually
reflects everything already published, without any code change to the home template
built in Epic 1 (NFR8).

## Acceptance criteria

1. One `article` record (`mode: external`, `platform: zenn`, `language: ja`) exists
   per piece of existing Zenn output the owner wants indexed (per spec §3's division
   of labor — Zenn stays canonical for JP; this site holds pointer records only).
2. Home's "Recent writing" block (Story 1.6, built empty-safe) now renders up to 5
   most-recent `article` records, mixed canonical/external, each with title, date,
   language badge, platform badge for external — with zero changes to Story 1.6's
   code (proving NFR8's additive-only guarantee held).
3. `/writing/` and `/feed.xml` reflect the new records automatically (Stories 2.2,
   2.4) — again, no code changes, only new content files (AC-2's "one new content
   file, no code changes" standard, applied here to prove it).

## Tasks / subtasks

- [ ] Author external-mode records for existing Zenn articles (owner supplies the
      list of URLs/titles/dates — do not fabricate).
- [ ] Verify home, `/writing/`, and `/feed.xml` all pick up the new records without
      any template changes.

## Dev notes

- This story is partly a verification step for Epic 1's additivity promise (NFR8) as
  much as it is content authoring — if any code change to Story 1.6's home template
  is needed here, that's a signal Story 1.6 wasn't built empty-safe and should be
  revisited, not a reason to special-case this story's content.
