# Story 1.5: Publication content type, Publications index, Publication detail

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.1

## Story

As a fellow researcher or prospective collaborator, I want to browse publications by
year and open a publication's full record, so that I can find the paper, its code,
and its citation.

## Acceptance criteria

1. `publication` records validate against spec §6.2 (`slug`, `title`, `authors`,
   `venue`, `year`, `status`, `links` — paper+code pairing enforced editorially, not
   by the build, since "code exists" isn't machine-checkable — `citation`, optional
   `featured` per D-11, optional `related`).
2. `/publications/` renders grouped by year, newest first; each entry: title, authors
   (owner highlighted), venue+year, links row, status badge for
   `preprint`/`superseded`; entry links to detail page (spec §7.3).
3. `/publications/<slug>/` renders: title, authors, venue, links (paper+code
   emphasized as a pair), abstract/summary body, BibTeX block, related entities,
   standard CTA block.
4. `superseded` status renders with a pointer to the superseding record (§6.4) —
   exercised with a fixture if no real superseded publication exists yet (AC-8).
5. SEO metadata + `ScholarlyArticle` JSON-LD present (FR7, spec §8.2).

## Dev notes

- The spec requires paper+code pairing "when code exists" (§6.2) — this is an
  editorial/authoring-time rule (see `docs/content-guide.md`), not something the
  build can enforce mechanically, so no validator check is added for it beyond what
  Story 1.1 already covers (required `links` array, ≥1 entry).
