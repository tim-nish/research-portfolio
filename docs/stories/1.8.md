# Story 1.8: SEO metadata, structured data, sitemap

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.1

## Story

As the site owner, I want every page to carry correct, record-derived SEO metadata and
structured data, and a generated sitemap, so that search engines index the site
correctly with zero hand-maintained per-page metadata (FR7, FR8).

## Acceptance criteria

1. A shared metadata component/function produces, for every page: unique `<title>`
   (pattern `<Page/Entity> — <Owner name>`; home is `<Owner name> — <positioning
   fragment>`), meta description from the record's `summary` (or profile field for
   non-entity pages), canonical URL tag, Open Graph + Twitter card tags (spec §8.2).
2. JSON-LD emitted per page type per spec §8.2's mapping: `Person` (home/about);
   `SoftwareApplication` (products — stubbed until Epic 3); `ScholarlyArticle`
   (publications); `Article` (canonical articles — stubbed until Epic 2);
   `Dataset`-bearing `CreativeWork` (benchmark projects with dataset links).
3. `/sitemap.xml` is generated from the content registry at build time, listing every
   indexable page that exists at build time (naturally grows as Epics 2–3 add
   content types, no code change needed then).
4. `robots.txt` is permissive (spec §8.2).
5. No page has hand-written per-page metadata overrides that bypass this system.

## Tasks / subtasks

- [ ] Build the shared `PageMeta`/JSON-LD component used by every page template from
      Stories 1.2–1.6.
- [ ] Build the sitemap generator as a build-time step alongside content validation
      (Story 1.1) and static generation.
- [ ] Add `robots.txt`.
