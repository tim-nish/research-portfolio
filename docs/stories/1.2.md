# Story 1.2: Profile content type + About page

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.1

## Story

As a visitor arriving via name search (P2, journey J3), I want an `/about/` page with
the owner's full bio, focus areas, and identity links, so that I can form a coherent
picture of expertise and find a contact path.

## Acceptance criteria

1. `content/profile.md` (singleton) validates against the `profile` schema (spec
   §6.2): `name`, `positioning`, `bio`, `now`, `focus_areas`, `identity_links`
   (ordered), `languages_note`.
2. `/about/` renders: full bio, focus areas, the two-track framing (research + tools)
   in the owner's voice, identity links, contact (email primary), languages note
   (spec §7.3 "About").
3. Page carries SEO metadata per FR7 (title pattern `<Page> — <Owner name>`,
   description from an appropriate profile field, canonical tag, OG/Twitter) and
   `Person` JSON-LD (spec §8.2).
4. Page is readable with JavaScript disabled (NFR1).
5. Empty optional blocks (e.g. `now` absent) render nothing, not an empty shell
   (AC-1).
