# Story 5.5: Canonical URL handling — deployment-matched canonical base

Canonical discussion record: https://github.com/tim-nish/website/issues/69
(part of umbrella #64 — canonical-base amendment, owner decision record
2026-07-18: tim-nish.dev is canonical; the Pages URL is hosting only). On
conflict, the issue wins.

As the site owner, I want every article page to declare its canonical URL
from the configured canonical base, so that search engines always see
tim-nish.dev as canonical regardless of the deployment host.

## Acceptance criteria

1. **Given** any article page (both modes' index entries and each
   site-canonical page), **when** built, **then** it emits `rel=canonical`
   built from the configured canonical base — tim-nish.dev — never from the
   deployment host URL.
2. **Given** the canonical base, **when** templates reference it, **then**
   it is a single config value and no page template hardcodes a host.
3. **Given** an external-mode record, **when** built, **then** its own
   external canonical URL is untouched.
4. **Given** a built site served from the GitHub Pages URL, **when**
   inspected, **then** it still declares tim-nish.dev canonicals
   throughout.
