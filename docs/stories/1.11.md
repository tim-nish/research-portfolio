# Story 1.11: Seed content migration & Phase 1 cutover

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.1–1.10 (all Epic 1 stories)

## Story

As the site owner, I want the current site's content migrated into the new registry
and the old hardcoded page retired, so that the live site is the Phase 1 hub, not the
single-page app it replaces.

## Acceptance criteria

1. `content/profile.md` is authored from the current `App.tsx` profile text plus
   `profile.identity_links` seeded with GitHub (`tim-nish`), LinkedIn (`nowwest`),
   and email; Hugging Face/Zenn/X left as commented placeholders per spec D-7
   (Appendix A).
2. `content/projects/kagamios.md` authored: `kind: system` (per Appendix A's explicit
   resolution of the plugin-vs-system ambiguity — already decided in the spec, not
   re-litigated here), `status: active`, summary from the current card, GitHub link.
3. `content/projects/quantscenariobench.md` authored: `kind: benchmark`,
   `status: active`, `featured: true`, summary + three links (GitHub, HF leaderboard,
   HF datasets) from the current card; `citation` added if available, otherwise
   omitted (optional field).
4. `src/App.tsx`'s hardcoded profile/project markup is removed; the app shell now
   routes to the generated pages from Stories 1.2–1.10.
5. Full Phase 1 acceptance sweep: AC-1, AC-2, AC-3, AC-4, AC-5, AC-6, AC-7, AC-9 hold
   (spec §13); AC-8 holds at least for the `archived` project state (fixture if no
   real archived project exists).
6. `npm run build` (per `CLAUDE.md`) passes clean: type-check + content validation +
   static generation, with no console errors, before this story is called done.
7. Performance/accessibility budgets (NFR4, NFR5 — spec §8.5) are checked and recorded
   against at least the home page and one representative entity detail page: total
   transfer under ~200 KB (excluding embedded demo media, no external font/CDN
   dependency), semantic landmarks present, full keyboard operability, image alt text
   present, WCAG AA contrast. This is the one point in Phase 1 these NFRs are actually
   gated, not just aspirational — record results (tool used, pages checked, pass/fail)
   in the story's completion notes.

## Tasks / subtasks
