# Story 1.6: Home page (hub)

**Status**: Draft
**Epic**: 1 — Hub Foundation
**Depends on**: 1.2, 1.3, 1.5, 1.7 (newsletter capture component)
**Build-order note**: this story's numeric ID is lower than 1.7's, but its AC1 needs
1.7's shared capture component. **Build 1.7 before 1.6** — story numbers in this epic
are IDs, not a mandated sequence; the "Depends on" line is authoritative.

## Story

As any visitor (P1–P5), I want the home page to communicate identity in one screen and
surface featured work, so that I immediately understand who this is and can go deeper
or subscribe (journey J3).

## Acceptance criteria

1. `/` renders, in order, exactly the seven blocks of spec §7.3 "Home": hero (name +
   `profile.positioning` + identity links); now (opt, `profile.now`); featured work
   (up to 3 records with `featured: true` across project/product/publication, curated
   order); recent writing (up to 5 most-recent `article` records — **stub/empty until
   Epic 2 ships the `article` type**; block renders nothing until then, per AC-1's
   empty-optional-block rule); sections directory (Projects/Benchmarks/Publications/
   Products/Newsletter, one-line each with count or micro-description); newsletter
   capture (primary placement, above footer); global footer.
2. "Featured work" draws from `project`, `product`, and `publication` records; since
   `product` doesn't exist until Epic 3, only `project`/`publication` `featured: true`
   records are eligible until then — this must not require touching this story's code
   again when Epic 3 lands (NFR8): the featured-query is type-agnostic from the start.
3. Curated order for featured items follows `docs/architecture.md` D-11's resolution:
   manual curation where present, else recency descending by each type's own date
   field.
4. Page readable with JS disabled; SEO metadata + `Person` JSON-LD present.
