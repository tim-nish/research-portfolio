# Story 2.1: Article content type (canonical + external)

**Status**: Draft
**Epic**: 2 — Writing System
**Depends on**: 1.1 (schema pipeline already defines this schema; this story
authors real records and confirms the mode-specific validation rule)

## Story

As the site owner, I want an `article` content type supporting both canonical
(hosted here) and external (pointer) modes, so that all writing — English long-form
and Zenn/JP pieces — is registered in one place without duplicating canonical
content (AP-6, FR12).

## Acceptance criteria

1. `article` records validate against spec §6.2: `slug`, `title`, `date`, `mode`
   (`canonical`|`external`), `language` (`en`|`ja`), `summary`; `external` (required
   when `mode: external`: `href`, `platform`); `syndication` (optional, canonical
   only); `topics`, `related`, `updated` (optional).
2. A `mode: canonical` record without a body fails the build; a `mode: external`
   record with a body fails the build (this specific rule was already scaffolded in
   Story 1.1 — this story adds the fixture/test proving it against real article
   content, if not already covered).
3. At least one real canonical and one real external record exist and validate
   (feeding Stories 2.2/2.3/2.5).
